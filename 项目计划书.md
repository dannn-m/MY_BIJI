# SDN 网络智能管理与安全防护系统 — 项目计划书

## 一、项目概述

### 1.1 项目背景

随着网络规模持续扩大，传统网络架构面临配置复杂、运维困难、安全防护被动等挑战。软件定义网络（SDN）通过控制平面与数据平面分离，为网络管理提供了集中化、可编程的全新范式。然而，现有 SDN 管理平台普遍存在以下不足：

- **拓扑感知局限**：大多工具仅展示设备连接关系，缺乏层次化、多维度的网络可视化
- **安全防护被动**：DDoS 检测与缓解往往分离，缺乏从检测到自动下发的闭环体系
- **流量模拟缺失**：测试环境中缺少真实业务流量，影响检测策略验证
- **管理粒度粗糙**：交换机和主机信息分散，缺乏统一的设备级管理视图

针对上述问题，本项目设计并实现了一套基于 OpenDaylight 控制器的 SDN 网络智能管理与安全防护系统。

### 1.2 项目目标

构建一个集**拓扑可视化、流量监控、DDoS 检测与自动缓解、设备管理、正常流量模拟**于一体的 SDN 综合管理平台，为网络管理员提供直观、智能、安全的网络运维体验。

### 1.3 适用范围

- 高校网络实验室教学演示
- 企业 SDN 网络运维管理
- 网络安全竞赛与攻防演练平台
- SDN 技术研究与性能评估

### 2.3 三层网络拓扑

系统模拟了一个典型的企业三层网络架构：

```
        L0 服务器层: s9 ─── aaa, dhcp, server (10.0.0.10-12)
                        │
        L1 核心层:   s7 ─── s8 ─── r10 (边界路由 10.0.0.254)
                     ╱ ╲    ╱ ╲
        L2 汇聚层:  s5     s6
                   ╱ ╲    ╱ ╲
        L3 接入层: s1 s2  s3 s4
                   │  │   │  │
        L4 终端层: h1 h2  h3 h4 h5 (10.0.0.1-5)
```

## 三、功能模块设计

### 3.1 系统仪表盘

- **网络状态概览**：实时展示交换机数量、主机数量、流表条目、链路数量
- **微型拓扑预览**：vis-network 渲染完整网络拓扑，支持双击跳转设备详情
- **系统健康监控**：ODL 控制器、Mininet 虚拟机、数据库连接状态
- **快捷导航**：一键跳转拓扑、设备、流量、日志等模块

### 3.2 拓扑可视化

- **层次化布局**：基于 vis-network 物理分层算法，自动排列 L0~L4 五层设备
- **设备差异化渲染**：交换机按层级（L0/L1/L2/L3）使用不同色调 SVG 图标；服务器、路由器、终端主机各有独立图标
- **智能悬停 Tooltip**：鼠标悬停交换机展示 DPID、端口数、流表数、总流量、连接主机 IP 等详细信息，API 数据缓存 30 秒
- **双击导航**：双击任意交换机跳转至设备详情页
- **动态拓扑刷新**：从 ODL 动态获取链路数据，结合预设拓扑补全完整网络视图

### 3.3 统一设备管理

- **自适应设备详情页**：同一页面模板根据设备类型（交换机/路由器/主机/服务器/防火墙）自动展示不同信息面板
- **交换机视图**：基本信息（DPID/层级/状态）+ 端口列表 + 连接主机 + 流表条目（支持增删）+ 端口流量统计
- **路由器/主机视图**：IP 地址、上联交换机、设备角色、可扩展属性
- **流表管理**：Web 端查看和删除流表条目，支持 ODL RESTCONF 与 SSH ovs-ofctl 双重删除策略
- **类型可扩展**：新建设备类型仅需在 `get_device_detail()` 中增加分支，前端自适应渲染

### 3.4 流量监控

- **实时流量采集**：每 10 秒通过 ODL RESTCONF 轮询各交换机端口 RX/TX 字节计数器
- **多维度统计**：按交换机、按端口、按时间范围（全部/5分钟/15分钟/1小时）筛选
- **Chart.js 可视化**：折线图展示流量趋势，交换机排名侧边栏
- **历史数据管理**：MySQL 持久化存储，每日凌晨自动清理 7 天前数据

### 3.5 DDoS 检测与缓解

#### 3.5.1 检测引擎

| 检测方法      | 算法              | 阈值              | 检测目标                  |
| --------- | --------------- | --------------- | --------------------- |
| 源 IP 熵值分析 | 归一化香农熵          | 熵值 < 0.5        | SYN Flood / UDP Flood |
| 目的端口熵值    | 归一化香农熵          | 熵值 < 0.3        | 端口扫描 / DNS 放大         |
| 流量速率检测    | 滑动窗口统计          | > 500 flows/s   | 大流量 DDoS              |
| 协议分布分析    | TCP/UDP/ICMP 占比 | TCP>70%/UDP>70% | 分类攻击类型                |

#### 3.5.2 缓解引擎

- **黑洞模式（Blackhole）**：在攻击侧接入交换机（s3/s4）和汇聚交换机（s6）下发 priority=65535 的 DROP 流表
- **牵引清洗模式（Redirect）**：将攻击流量通过流表重定向至清洗容器，清洗后回注正常流量
- **自动缓解**：检测告警后自动部署，支持手动触发和规则回滚
- **多路径下发**：ODL RESTCONF 优先，SSH ovs-ofctl 兜底，确保流表部署可靠性

#### 3.5.3 告警管理

- 告警去重冷却（60 秒内同类型不重复）
- 四级严重程度：low / medium / high / critical
- 告警历史查询与活跃告警面板
- 缓解规则生命周期管理

### 3.6 正常流量模拟

- **随机打流引擎**：每 20 秒从 h1~h5 中随机选择 2-3 台主机
- **混合流量模式**：35% ping 探测 + 35% TCP SYN + 20% 突发通信 + 10% HTTP 探测
- **目标选择**：随机选择其他主机或 server(10.0.0.11)，服务器和路由器不主动发起流量
- **SSH 执行**：通过 `sudo mnexec` 进入 Mininet 主机网络命名空间执行
- **完全无 Scapy 依赖**：仅使用 Python socket + subprocess ping，轻量级部署

### 3.7 用户权限管理

- **角色体系**：管理员（admin）和普通用户（user）
- **功能**：登录/注册、密码强度校验、管理员可管理用户（启用/禁用/删除/改角色）
- **安全**：CSRF 保护、密码哈希存储、会话管理

## 六、创新特色

### 6.1 自适应的统一设备管理

传统 SDN 管理工具通常将交换机、主机、路由器分开管理。本系统设计了**统一设备详情页**，通过设备类型自动识别，同一页面模板自适应展示不同信息：交换机显示端口/流表/主机连接，路由器显示 IP/上联/路由策略，服务器显示服务类型/连接拓扑。新增设备类型仅需后端添加一个分支，前端零修改。

### 6.2 三重流表操作保障

流表的增删操作采用**三重策略**确保可靠性：
1. ODL RESTCONF Config Store（标准方式）
2. ODL RESTCONF Operational Store（兼容 ODL 内部流表）
3. SSH ovs-ofctl 直接操作（终极兜底）

当控制器 API 不可用时，系统自动降级为直接交换机操作，保证管理功能不中断。

### 6.3 DDoS 检测-缓解闭环

系统构建了完整的 **sFlow 采样 → 熵值检测 → 攻击分类 → 自动缓解 → 效果验证** 闭环：
- sFlow v5 采集器实时接收交换机采样数据
- 多维度熵值分析识别异常流量模式
- 自动分类攻击类型（SYN Flood / UDP Flood / DNS Amplification 等）
- 在攻击侧交换机自动下发高优先级 DROP 流表
- 支持黑洞和牵引清洗两种缓解策略

### 6.4 混合模式正常流量模拟

解决了 SDN 测试环境中**缺乏真实业务流量**的痛点：
- 无需 Scapy 等第三方依赖，纯 Python stdlib 实现
- 70% 随机突发 + 30% 轻量探测，模拟真实用户行为
- 流量经过 OVS 交换机产生 sFlow 采样，形成完整数据链路
- 为 DDoS 检测引擎提供对比基线

### 6.5 层次化拓扑可视化

基于 vis-network 物理分层引擎，将 9 台交换机、5 台主机、3 台服务器、1 台路由器按五层架构自动排列，支持智能悬停详情、双击导航、实时刷新。

## 七、部署方案

### 7.1 环境要求

| 组件 | 要求 |
|------|------|
| 操作系统 | Windows 10+ / Ubuntu 20.04+ |
| Python | 3.8+ |
| MySQL | 5.7+ |
| OpenDaylight | Nitrogen 0.7+ (含 RESTCONF 模块) |
| Mininet | 2.3+ (含 Open vSwitch) |
| 浏览器 | Chrome 90+ / Edge 90+ |

### 7.2 部署步骤

```
1. 启动 Mininet 网络拓扑（sudo python3 sdntopo.py）
2. 启动 OpenDaylight 控制器（./bin/karaf）
3. 安装 ODL 功能：feature:install odl-restconf odl-l2switch-switch
4. 创建 MySQL 数据库：CREATE DATABASE sdn_monitor
5. 配置 .env 环境变量（ODL_IP, MININET_VM_IP, DATABASE_URL 等）
6. 初始化数据库：python init_db.py
7. 启动 Web 应用：python app.py
8. 浏览器访问 http://localhost:5000
```
