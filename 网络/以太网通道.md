# 一、概念
## 1.
是一种lan间的交换机间技术，可以把多个快速以太网或者千兆以太网端口划分到一个逻辑通道中
![[Pasted image 20260415153007.png]]
## 2.优点
- 大多配置任务可以在一个通道接口完成（不是成员端口），可以确保链路中的配置一致
- 无需升级物理链路
- 负载均衡
- 提供冗余
## 3.限制
- 不能混用快速端口和千兆口
- 最多可以由8个配置兼容的端口组成，提供最高800mb/s或8gb/s（千兆口）
- 在两交换机间的成员端口配置需要相同（中继、本征vlan...)
- 在逻辑接口的配置应用于所有物理接口
## 4、自动协商协议实现
- 可以两个协议之一协商建立，端口聚合协议（PAgP）与链路控制聚合协议（LACP），协议允许交换机具有相似特征的端口协商为通道
**注意**：可以通过配置静态或者无条件以太通道，不使用协商
# 二、工作原理
## 1.PAgP
数据包每30s发送一次用于检查端口配置一致性
有3种模式如下：
- on-强制不适用pagp建立通道（直接建立）此模式不会交换数据包
- PAgP desirable-主动协商
- PAgP auto-被动协商（与ospf端口模式类似）
**匹配模式**：

|S1|S2|是否能建立通道|
|:--|:--|:--|
|On|On|是|
|On|Desirable/Auto|否|
|Desirable|Desirable|是|
|Desirable|Auto|是|
|Auto|Desirable|是|
|Auto|Auto|否|
## 2.LACP
有三种模式（功能与PAgP类似)：
- on
- LACP active
- LACP passive

| S1      | S2             | 是否能建立通道 |
| :------ | :------------- | :------ |
| On      | On             | 是       |
| On      | Active/Passive | 否       |
| Active  | Active         | 是       |
| Active  | Passive        | 是       |
| Passive | Active         | 是       |
| Passive | Passive        | 否       |
# 三、配置指南
## 1、原则
- **接口支持**：所有成员接口必须支持以太通道，无需物理端口连续。
- **速率双工一致**：同一条通道内所有**接口的速率、双工模式**必须完全相同。
- **VLAN 匹配**：所有接口需分配至同一 VLAN，或统一配置为中继端口。
- **VLAN 范围一致**：中继通道的所有接口，允许通过的 VLAN 范围必须一致；范围不匹配时，即使协商模式正确也无法建立通道。
## 2.LACP配置示列
**默认情况以太通道处于禁用状态**
1. 全局命令模式下**interface range**_interface_组成以太通道的接口
2. 在接口范围配置模式下**channel-group** identifer **mode active**创建端口通道接口，标识符指定了通道组的编号。mode active指定这是一条LACP以太通道
3. 要更改端口通道上的2层设置，需使用interface port-channel命令，后跟接口标识符，进入端口通道接口配置模式

```
S1(config)# **interface range FastEthernet 0/1 - 2**

S1(config-if-range)# **channel-group 1 mode active**

Creating a port-channel interface Port-channel 1

S1(config-if-range)# **exit**

S1(config)# **interface port-channel 1**

S1(config-if)# **switchport mode trunk**

S1(config-if)# **switchport trunk allowed vlan 1,2,20**
```
